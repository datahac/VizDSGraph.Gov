## This is copmlete project's schema with comments on coding 

# Nodes with explanation

Feature - A functional capability or component implemented within a project system that operationalizes tasks such as consent management, bias monitoring, audit logging, or reporting tools.
Role - A stakeholder interacting with or infrulencing the system or governance process (e.g., clinician, founder, ethical board member). Represents role groups rather than specific individuals. Specific to a particular project. 
_Neodash_Dashboard - Techical node. Do not use.
Dataset - A dataset used or generated by a project. Captures data source type, structure, identifiers, and governance characteristics but not individual records.
TopLevelProblem - A high-level decision or challenge faced by stakeholders that can be explored through visual analytics.
DetailedProblem - A specific decomposition of a TopLevelProblem that can be operationalized as a query or visual analytic exploration.
Project - A specific HealthTech AI project being analysed within the governance knowledge graph.
Regulation - A legal or regulatory framework (e.g., EU AI Act, GDPR) that defines compliance obligations relevant to projects.
RegulatoryRequirement - A specific actionable obligation derived from a regulation that a project must address.
Hub - An innovation hub, accelerator, or organisational network linking projects and providing support or infrastructure.
Module - A major AI technical subsystem or architectural component of the project system. This is often key to the Project, and constitutes the AI innovation by the project. 
EvidenceArtifact - A document, report, dataset, or technical output demonstrating compliance, evaluation, or governance evidence.
IntendedUse - The declared purpose or application context of the AI system (e.g., clinical decision support).
Funder - An organisation or funding source providing financial support to a project.
RoleType - Category of roles


# Edges with explanation

FromNode	Relationship	ToNode	Explanation
[Dataset]	PART_OF_PROJECT	[Project]	Indicates that the dataset belongs to or is used within a specific project context. The dataset is considered part of the project’s data resources but does not represent the project itself.
[EvidenceArtifact]	PART_OF_PROJECT	[Project]	Indicates that the evidence artifact was produced or used within the project and forms part of the project’s governance, documentation, or validation evidence.
[Feature]	FULFILLS	[RegulatoryRequirement]	Indicates that a specific system feature operationalizes or helps satisfy a regulatory requirement. The feature provides a technical or procedural mechanism enabling compliance.
[Feature]	FulfilledBy	[TopLevelProblem]	Indicates that the feature exists in order to address or resolve a high-level analytical or governance problem encountered by stakeholders.
[Feature]	Uses	[Feature]	Indicates a dependency or compositional relationship where one feature relies on another feature for implementation or functionality.
[Funder]	FUNDS	[Project]	Indicates that the funder provides financial support for the project, applies project de-risking assessment, and impacts project obligations. The relationship captures funding origin but not the amount or contract details.
[Hub]	LINKS_TO	[Project]	Indicates that the project is associated with or supported by an innovation hub, accelerator, or organisational ecosystem.
[IntendedUse]	REQUIRES	[RegulatoryRequirement]	Indicates that a given intended use of a system triggers or necessitates specific regulatory requirements that must be addressed.
[Module]	PART_OF_PROJECT	[Project]	Indicates that the module is an AI subsystem within the project architecture.
[Project]	HAS_INTENDED_USE	[IntendedUse]	Indicates the declared purpose or operational context for which the project’s system is designed, from the perspective of Health AI regulations.
			
[Project]	PROJECT_HAS_FEATURE	[Feature]	Indicates that the project implements or includes a particular system capability or feature.
[Project]	PROJECT_HAS_ROLE	[Role]	Indicates that a stakeholder role is involved in the project, representing a category of actors interacting with or influencing the project.
[Project]	PROJECT_SUBJECT_TO_REGULATION	[Regulation]	Indicates that the project operates under the scope of a regulatory framework and must comply with it.
[Regulation]	Needs	[TopLevelProblem]	Indicates that the regulation introduces governance challenges or analytical questions that projects must address to demonstrate compliance.
[Regulation]	REQUIRES	[RegulatoryRequirement]	Indicates that the regulation consists of specific actionable requirements that must be satisfied.
			
[Role]	Needs	[TopLevelProblem]	Indicates that the stakeholder role seeks to answer or resolve the specified high-level analytical problem.
[Role]	Uses	[Feature]	Indicates that the stakeholder role interacts with or relies on a specific system feature to perform tasks.
[TopLevelProblem]	Decompose	[DetailedProblem]	Indicates that a high-level problem is broken down into more specific analytical questions suitable for investigation.
[TopLevelProblem]	FulfilledBy	[Feature]	Indicates that resolving the high-level problem depends on the presence or implementation of specific system features.
[Role]	ROLE_INSTANCE_OF	[RoleType] Inticates what generic RoleType a particular project-related Role is an instance of.

# Parameters with explanation, Node	Property	Explanation

Dataset columns List of variables or fields contained in the dataset. Used to characterise dataset content, support assessment of analytical potential, and inform visualisation, modelling, and integration decisions.

Dataset contains_pii Indicates whether the dataset includes personally identifiable information. Used to assess privacy risk and regulatory obligations such as GDPR compliance.

Dataset data_structure Specifies the structural form of the dataset (e.g., tabular, time series, graph, text, multimodal). Used to guide appropriate storage, querying, preprocessing, and analysis methods.

Dataset dataset_id Unique identifier for the dataset within the knowledge graph to ensure consistent referencing across projects and analyses.

Dataset modalities Describes the types of data represented in the dataset (e.g., clinical, sensor, imaging, text). Used to capture data heterogeneity and support reasoning about multimodal integration.

Dataset name Human-readable name of the dataset used or generated by the project.

Dataset num_attributes Number of variables or features contained in the dataset. Used to estimate dataset dimensionality and analytical complexity.

Dataset num_records Number of records, rows, or observations contained in the dataset. Used to estimate dataset scale and support methodological and infrastructure decisions.

Dataset record_unit Defines the primary entity represented by each record in the dataset (e.g., patient, event, trial, organisation). Used to clarify analytical granularity and support alignment across datasets.

Dataset source_type Describes the origin or provenance of the data source (e.g., synthetic, clinical records, wearable sensors, surveys, imaging). Used to assess data realism, collection context, and downstream methodological implications.
